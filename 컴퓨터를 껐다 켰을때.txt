Hyperledger Fabric 테스트 네트워크 상태 재확인 (가장 중요):
이전에 성공했을 때와 마찬가지로, fabric-samples/test-network 디렉토리에서 Hyperledger Fabric 테스트 네트워크의 모든 Docker 컨테이너(특히 peer0.org1.example.com, orderer.example.com, ca_org1)가 정상적으로 Up 상태인지 다시 한번 꼼꼼히 확인해야 합니다.
확인 방법: docker ps -a 명령어를 실행합니다. (제공해주신 Docker Desktop 스크린샷에는 dev-peer0.org1.example.com... 이미지가 보이긴 하지만, 이것이 현재 실행 중인 컨테이너의 Up 상태를 의미하지는 않습니다. docker ps로 실행 상태를 확인해야 합니다.)
만약 컨테이너들이 Exited 상태이거나 보이지 않는다면, 네트워크가 제대로 시작되지 않았거나 중지된 것입니다.
네트워크 재시작 (가장 확실한 해결책 중 하나):
문제가 지속된다면, 가장 확실한 방법은 Fabric 네트워크를 완전히 종료했다가 다시 시작하는 것입니다. Docker 컨테이너들이 시간이 지나면서 예기치 않게 중단되거나 네트워킹에 문제가 생길 수 있습니다.
